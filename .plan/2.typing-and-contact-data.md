## 2. Типизация и контракты данных

- [ ] **Базовый тип сущности**

  ```ts
  export interface EntityItem {
    id: string; // crypto.randomUUID()
    name: string;
    date: string; // ISO 8601 для совместимости с AntD DatePicker
    value: number;
  }
  ```

- [ ] **Типы для формы и валидации**
  - [ ] `EntityFormValues`: `Omit<EntityItem, 'id'> & { id?: string }`
  - [ ] Использовать **Zod** для схемы валидации (интеграция с AntD Form через `zodResolver` или ручная обёртка)
  - [ ] Типизировать ошибки: `Record<keyof EntityFormValues, string | undefined>`

- [ ] **Типы для таблицы AntD**
  - [ ] `ColumnType<EntityItem>` с явным `sorter`: `CompareFn<EntityItem>`
  - [ ] `SortOrder` из `antd/es/table/interface`
  - [ ] Утилита для безопасного доступа к полям: `type EntityField = keyof EntityItem`
